Réinitialisation Trésorerie

Réinitialiser les données de trésorerie
Supprime uniquement les données de trésorerie : imports FEC, comptes détectés, données mensuelles, solde initial et ajustements Budget N.
Les autres données de l'application (utilisateurs, saisies, congés, etc.) ne sont pas affectées.

Vous pourrez ensuite réimporter vos fichiers FEC depuis la page Trésorerie.

Zone de danger

Reinitialiser la base de donnees
Supprime toutes les donnees de l'application : utilisateurs, saisies, conges, plannings, subventions, budgets, etc. La structure sera recree a vide et vous serez redirige vers la creation d'un nouveau compte administrateur.

Cette action est irreversible. Une sauvegarde de securite sera creee automatiquement.

Guide de gestion des mises a jour

Le systeme de migrations permet de mettre a jour la structure de la base de donnees de maniere controlee et securisee.

  • Sauvegardez toujours avant d'appliquer une mise a jour (menu Administration > Sauvegardes)
  • Appliquez dans l'ordre : les migrations sont numerotees et doivent etre appliquees sequentiellement
  • Verifiez le statut : la page indique si des mises a jour sont en attente
  • En cas de probleme : restaurez la sauvegarde depuis la page Sauvegardes
  • Premiere utilisation : si la base existe deja, utilisez le bouton "Marquer le schema initial" pour demarrer le suivi
